# Renewlet v0.1.5

Renewlet v0.1.5 focuses on calendar handling, statistics, public status pages, and self-hosting reliability. Individual calendar downloads, future cost trends, and public-page theme switching are now covered, with steadier import, version, and Cloudflare scheduled task behavior.

[中文 ->](./v0.1.5-zh.md)

---

## Read Before Upgrading

No manual migration is required when upgrading from v0.1.4. Docker users can still open System Update from the version badge at the top of Renewlet. Cloudflare users can sync their fork and use the normal deployment flow; D1 migrations run before the Worker is published.

---

## New Features

- Individual subscription calendar download: Add to Calendar can download a one-off `.ics` file for calendar clients that do not use webcal subscription links.
- Cost trends: the statistics page shows expected spending for the next 12 months, with views for future charges and amortized monthly cost ([#26](https://github.com/zhiyingzzhou/renewlet/issues/26), thanks [@cnrot](https://github.com/cnrot)).
- Public status page theme switching: public-link visitors can choose light, dark, or system mode without signing in.

---

## Improvements

- The version badge is now visible to all signed-in users. In-app update, restart, and diagnostic error details remain admin-only.
- The header falls back to the frontend build version while the version API is loading, avoiding a briefly empty version badge.
- Built-in icon source checks now run when the main source dialog opens. Provider popovers keep only status details and manual refresh.

---

## Bug Fixes

- Fixed Add to Calendar possibly showing a temporary page error after adding a calendar entry ([#22](https://github.com/zhiyingzzhou/renewlet/issues/22), thanks [@hkl8080](https://github.com/hkl8080)).
- Fixed uploaded icon lists staying stale after importing data with Logo assets.
- Fixed Cloudflare subscription updates failing on historical dirty `tags_json` values; saves now normalize them back to a valid tag array.
- Cloudflare icon uploads now reject obviously oversized requests earlier instead of sending large files through form parsing first.
- One-off `.ics` downloads no longer include refreshable feed fields that could make calendar clients treat them as subscription feeds.

---

## Privacy and Reliability

- One-off `.ics` downloads require a signed-in session, do not expose public feed tokens, and include `no-store` and `nosniff` headers.
- Non-admin users can read version status without receiving raw GitHub upstream error details.
- Cloudflare scheduled tasks now run auto-renewal, notifications, and cloud backups as separate phases. If one phase fails, Renewlet logs a redacted summary and continues with the remaining phases.
- Built-in icon source checks still access GitHub feeds sequentially to reduce 403/429 risk on shared egress.
