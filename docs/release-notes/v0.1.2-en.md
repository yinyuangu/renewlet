# Renewlet v0.1.2

Renewlet v0.1.2 adds cloud backups and restore, built-in icon source status, notification email refinements, and clearer Cloudflare deployment behavior, while fixing AI recognition, login redirects, settings, and mobile reliability issues.

[中文 ->](./v0.1.2-zh.md)

---

## Read Before Upgrading

No manual migration is required when upgrading from v0.1.1. Docker release images can continue to update from the version badge at the top of Renewlet. For Cloudflare deployments, sync your fork and make sure D1 migrations run before the Worker is published.

Cloud backups require a WebDAV or S3-compatible storage target in Settings. When a connection fails, Renewlet shows redacted upstream responses and diagnostics so you can check permissions, endpoints, buckets, or remote paths.

The optional Cloudflare `RENEWLET_GITHUB_TOKEN` is now uploaded as a Worker Secret. You only need it when version checks or built-in icon source checks hit GitHub rate limits; core usage does not require it.

---

## New Features

- Cloud backups: configure WebDAV and S3-compatible targets, then create, list, download, delete, and restore backup snapshots.
- Backup target checks: connection tests cover write, read, delete, and list permissions so list-only failures are caught before use.
- Built-in icon source status: Settings now shows icon source, version, and update status so embedded/runtime icon indexes are easier to inspect.
- AI recognition diagnostics: provider errors now include richer status, response snippets, and redacted diagnostics to help fix model, endpoint, or key settings.

---

## Improvements

- Notification emails now use a quieter ledger-style template with preheaders, theme colors, reminder groups, and action buttons, without meaningless summaries for empty or test notifications.
- Settings now separates save and error states for cloud backups, AI settings, icon sources, and notification configuration, making retries more predictable.
- Setup, login, mobile overlays, time and numeric inputs, import preview, and subscription lists behave more reliably on small screens and failed API responses.
- Cloudflare deployment docs now cover GitHub rate limits, one-click deploy version display, local HTTP dev behavior, and optional Worker Secrets.

---

## Bug Fixes

- Fix PocketBase session refresh missing a request body in some tunnel or proxy environments, which could send users back to login ([#16](https://github.com/zhiyingzzhou/renewlet/issues/16)).
- Fix repeated subscription-list cursors that could trigger request loops across dashboard, statistics, calendar, and aggregate list views.
- Fix cached cloud-backup list errors leaking into inline copy after switching languages, while keeping raw payload details in the error dialog.
- Fix extra spacing caused by Popover, Select, and DropdownMenu portals inside the built-in icon source dialog.

---

## Security and Reliability

- Cloud backup remote errors redact access keys, passwords, signatures, and sensitive response details while preserving host, status, and error code diagnostics.
- S3-compatible storage supports both service endpoint and path-style endpoint modes, improving compatibility with R2, COS, MinIO, and similar services.
- The optional Cloudflare Worker GitHub token is deployed through a temporary secrets file instead of wrangler config or logs.
- Release and Docker in-app update paths still require Linux amd64/arm64 tarballs and `checksums.txt`; Cloudflare deployments continue to expose manual upgrade guidance only.
