# Renewlet v0.1.6

Renewlet v0.1.6 重点改善通知调度在自托管和 Cloudflare 部署中的后台性能。通知提醒仍按原有时间、渠道和内容发送，但大订阅量下的定时检查会更轻，Cloudflare D1 免费额度也更不容易被空跑的 cron 消耗掉。

---

## 升级前请先读

Docker 用户从 v0.1.5 升级不需要手动迁移。Cloudflare 用户请继续按现有部署流程发布；本版本包含用于通知调度的 D1 索引。

---

## 优化改进

- 通知定时任务现在会先判断是否进入用户本地提醒窗口，再读取可能需要提醒的订阅。大多数非提醒时间不再扫描当前用户的全部订阅。
- 重复提醒仍保持原有语义，但非日常提醒窗口只读取开启重复提醒的候选订阅，减少后台空转。
- 自动续订维护继续在通知内容生成前执行，避免已自动续订的订阅被错误地当作过期或即将续费提醒。
- 订阅详情弹窗的桌面按钮区更紧凑，编辑、续订、日历和关闭操作保持原有顺序与功能。

---

## 可靠性与性能

- Cloudflare 部署下，通知 cron 的 D1 读取量不再随订阅总数按分钟线性增长；订阅较多的账号会更明显受益。
- Docker/PocketBase 部署同样减少了每分钟后台任务对 SQLite/PocketBase 的无效读取，降低大数据量下的 I/O 和 CPU 压力。
