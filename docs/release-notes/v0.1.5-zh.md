# Renewlet v0.1.5

Renewlet v0.1.5 主要整理了日历、统计、公开状态页和自托管可靠性。单个订阅的日历下载、未来费用走势和公开页主题切换都补齐了，导入、版本展示和 Cloudflare 定时任务也更稳。

[English ->](./v0.1.5-en.md)

---

## 升级前请先读

从 v0.1.4 升级不需要手动迁移。Docker 用户可以继续从页面顶部版本号打开“系统更新”；Cloudflare 用户同步 fork 后按部署流程发布即可，D1 migrations 会先于 Worker 执行。

---

## 新增功能

- 单个订阅日历下载：在“添加到日历”里可以下载一次性 `.ics` 文件，适合不使用 webcal 订阅链接的日历客户端。
- 费用走势：统计页展示未来 12 个月预计支出，支持“未来扣费”和“月均摊销”两个视角（[#26](https://github.com/zhiyingzzhou/renewlet/issues/26)，感谢 [@cnrot](https://github.com/cnrot)）。
- 公开状态页主题切换：公开链接访问者可以切换浅色、深色或跟随系统，不需要登录。

---

## 优化改进

- 版本号 badge 现在对所有登录用户可见；页面内更新、重启和错误详情仍只开放给管理员。
- 版本接口加载前会使用前端构建版本兜底，避免顶部版本号短暂空白。
- 内置图标来源改为打开总弹层时批量检查，provider 弹层只保留详情和手动刷新。

---

## Bug 修复

- 修复“添加到日历”后可能提示页面暂时无法显示的问题（[#22](https://github.com/zhiyingzzhou/renewlet/issues/22)，感谢 [@hkl8080](https://github.com/hkl8080)）。
- 修复导入带 Logo 的数据后，上传图标列表可能继续显示旧缓存的问题。
- 修复 Cloudflare 订阅更新遇到历史脏 `tags_json` 时可能失败的问题；保存时会收敛回合法标签数组。
- Cloudflare 图标上传会更早拒绝明显超过限制的请求，避免大文件先进入表单解析流程。
- 一次性 `.ics` 下载不会再写入订阅 feed 的刷新字段，避免日历客户端误认为它是可刷新订阅。

---

## 隐私与可靠性

- 一次性 `.ics` 下载需要登录态，不暴露公开 feed token，并带有 `no-store` 和 `nosniff` 响应头。
- 非管理员可以查看版本状态，但不会收到 GitHub 上游原始错误详情。
- Cloudflare 定时任务按自动续订、通知、云备份分阶段执行；某一阶段失败会记录脱敏摘要，并继续后续阶段。
- 内置图标来源检查继续串行访问 GitHub feed，减少共享出口下触发 403/429 的风险。
