# Renewlet v0.1.4

Renewlet v0.1.4 主要整理了三件事：公开演示站、自定义外部脚本、Cloudflare 部署。版本检查和内置图标检查也顺手减负，不再要求额外配置 GitHub token。

[English ->](./v0.1.4-en.md)

---

## 升级前请先读

从 v0.1.3 升级不需要手动迁移。Docker 用户可以继续从页面顶部版本号打开“系统更新”；Cloudflare 用户同步 fork 后按部署流程发布即可，D1 migrations 会先于 Worker 执行。

Cloudflare 本版需要 `ASSETS` Static Assets binding。一键部署和 GitHub Actions 部署会使用新配置；手动 Wrangler 部署用户请重新生成配置后再发布。

`RENEWLET_GITHUB_TOKEN` 已经不再参与版本检查或内置图标检查。之前加过这个环境变量或 Worker Secret 的用户可以删掉。

`RENEWLET_CUSTOM_HEAD_SCRIPT` 默认关闭。Docker/Go 部署改完环境变量后重启 Renewlet 即可；Cloudflare Static Assets 需要在构建时带上这个变量，然后重新构建和部署。

---

## 新增功能

- 自定义 Head 外链脚本入口：`RENEWLET_CUSTOM_HEAD_SCRIPT` 只在部署者显式配置时注入一个外部 `<script>`。
- 在线演示入口：README 增加了公开演示站地址和演示账号，想先试一下可以直接打开。
- Demo 数据换了一批：演示订阅改成开发者工具、云服务、设计协作、可观测性、安全认证等真实场景，日期会跟着重置时间滚动。

---

## 优化改进

- 版本检查改读 GitHub Release Atom feed；Docker 页面内更新仍会检查 release tarball 和 `checksums.txt`，但不用再为这件事配 GitHub token。
- 内置图标 seed 索引改成搜索和详情两份 gzip 资产，Cloudflare 首次读取和图标检索更轻。
- 官网构建会按当前 GitHub Pages 地址或自定义域生成 canonical、OG image、robots 和 sitemap，仓库页和自定义域不容易串路径。
- Cloudflare 部署文档删掉了 GitHub token 相关步骤，secrets 列表更短。

---

## Bug 修复

- 修复官网在 GitHub Pages 自定义域或仓库子路径下，资源路径、分享链接和 sitemap URL 可能不一致的问题。
- 修复 Cloudflare release 发布时还在准备旧 Worker Secret 的问题。
- 自定义脚本现在只接受单个外链 script，并会拒绝内联脚本和事件属性；对应 CSP 也会一起更新。
- 修复内置图标来源弹窗打开、关闭后检查状态没有及时收尾的问题。

---

## 隐私与可靠性

- Renewlet 默认不注入任何外部脚本；只有部署者显式配置 `RENEWLET_CUSTOM_HEAD_SCRIPT`，才会加载其提供的外链 script。
- 隐私页补充了外部脚本说明：如果部署者启用这类脚本，访问数据可能会发送给脚本服务商。
- 版本检查、内置图标检查和 Cloudflare 部署都移除了 runtime GitHub token 路径，需要保管的 secret 更少。
- Demo Mode 继续跳过真实通知发送、云备份等外部操作，不会影响普通生产实例。
