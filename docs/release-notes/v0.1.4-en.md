# Renewlet v0.1.4

Renewlet v0.1.4 is mostly about the public demo, custom external scripts, and Cloudflare deployments. Version checks and built-in icon checks also no longer need a GitHub token.

[中文 ->](./v0.1.4-zh.md)

---

## Read Before Upgrading

No manual migration is required when upgrading from v0.1.3. Docker users can still open System Update from the version badge at the top of Renewlet. Cloudflare users can sync their fork and use the normal deployment flow; D1 migrations run before the Worker is published.

Cloudflare now needs the `ASSETS` Static Assets binding. One-click deployments and GitHub Actions deployments use the new config; manual Wrangler users should regenerate the config before publishing.

`RENEWLET_GITHUB_TOKEN` is no longer used for version checks or built-in icon checks. If you added this environment variable or Worker Secret before, you can remove it.

`RENEWLET_CUSTOM_HEAD_SCRIPT` is empty by default. Docker/Go deployments pick it up after a restart. Cloudflare Static Assets deployments need the variable at build time, followed by a rebuild and redeploy.

---

## New Features

- Custom head script entry: `RENEWLET_CUSTOM_HEAD_SCRIPT` injects one external `<script>` only when the deployer explicitly sets it.
- Public demo link: README now includes the demo URL and demo account, so you can try Renewlet before deploying it.
- New demo data: Demo Mode now uses real-world developer tools, cloud services, design tools, observability, and security/auth examples. Dates move forward after each reset.

---

## Improvements

- Version checks now read the GitHub Release Atom feed. Docker in-app updates still check the release tarball and `checksums.txt`, but this no longer needs a GitHub token.
- Built-in icon seed indexes are now split into search and detail gzip assets, so Cloudflare has less to read on first use.
- Website builds now use the current GitHub Pages URL or custom domain for canonical links, Open Graph images, robots.txt, and sitemap.xml.
- Cloudflare deployment docs dropped the GitHub token steps and have a shorter secrets list.

---

## Bug Fixes

- Fixed website asset paths, share links, and sitemap URLs that could get mixed up on GitHub Pages custom domains or repository subpaths.
- Fixed the Cloudflare release deployment still preparing an old Worker Secret.
- Custom scripts now accept only one external script, reject inline scripts and event handler attributes, and update CSP at the same time.
- Fixed the built-in icon source popover leaving status behind after open and close cycles.

---

## Privacy and Reliability

- Renewlet does not inject external scripts by default. It loads a deployer-provided external script only when `RENEWLET_CUSTOM_HEAD_SCRIPT` is explicitly set.
- The privacy page now mentions that deployer-provided scripts may send visit data to their provider.
- Version checks, built-in icon checks, and Cloudflare deployments no longer use the runtime GitHub token path, so there are fewer secrets to keep.
- Demo Mode still skips real notification sends, cloud backups, and other external actions. Normal production instances are unchanged.
