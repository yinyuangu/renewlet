# Renewlet v0.2.8

Renewlet v0.2.8 focuses on speed and reliability for Cloudflare deployments with larger subscription lists. Subscription lists, Public API totals, reminders, auto-renewal, and cloud backup schedules are steadier, and private logo loading feels smoother.

## Read Before Upgrading

Cloudflare one-click deploy users should still upgrade from the generated repository by running `Sync Renewlet Upstream`. Do not click the one-click deploy button again. If an older generated repository does not have this entry yet, add it once from the [Cloudflare deploy documentation](https://github.com/zhiyingzzhou/renewlet/blob/main/docs/cloudflare-workers-deploy.md#existing-one-click-deploy-users).

---

## Improvements

- Cloudflare subscription lists now respond faster for filters, search, tags, and totals. With larger lists, pagination and Public API totals stay more stable.
- Cloudflare notifications, auto-renewal, and cloud backup schedules use fewer resources by only handling accounts that need work now. Missed jobs after downtime or failed runs are still caught up.
- Private logos and icons are reused within the same signed-in session, reducing repeated requests and flicker when lists reflow, pages change, or dialogs open.
- Subscription list requests are cached briefly, so returning to a page or switching views does not reload the same data as often.
- The Statistics trend tooltip no longer repeats secondary metadata, making subscription details easier to scan.
