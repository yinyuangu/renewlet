# Renewlet v0.1.6

Renewlet v0.1.6 focuses on background notification performance for both self-hosted Docker installs and Cloudflare deployments. Reminder timing, channels, and message content stay the same, but scheduled checks are much lighter for accounts with many subscriptions, especially on Cloudflare D1.

[中文 ->](./v0.1.6-zh.md)

---

## Read Before Upgrading

Docker users do not need any manual migration when upgrading from v0.1.5. Cloudflare users should keep using the normal deployment flow; this release includes new indexes for notification scheduling.

---

## Improvements

- Scheduled notifications now check the user's local reminder window before reading subscriptions. Most non-reminder minutes no longer scan every subscription for that user.
- Repeat reminders keep the same behavior, but non-daily reminder windows now read only repeat-reminder candidates instead of the full subscription list.
- Auto-renewal maintenance still runs before notification content is built, so renewed subscriptions are not incorrectly treated as expired or due.
- The desktop subscription detail dialog now uses a more compact action area. Edit, renew, calendar, and close actions keep the same order and behavior.

---

## Reliability and Performance

- On Cloudflare, notification cron D1 reads no longer grow linearly with the total number of subscriptions every minute. Accounts with larger subscription lists should see the biggest reduction.
- Docker/PocketBase installs also avoid unnecessary per-minute SQLite/PocketBase reads, reducing background I/O and CPU pressure on larger datasets.
